## ūüöÄ Why pycall-node?

Most Node ‚ÜĒ Python bridges **spawn a fresh process on every call** ‚ÄĒ adding hundreds of milliseconds of overhead. `pycall-node` keeps **one persistent subprocess** alive and communicates over a raw stdio NDJSON pipe.

```
‚ĚĆ  Traditional:   Node ‚Üí spawn() ‚Üí python ‚Üí result ‚Üí kill()   (~300ms per call)
‚úÖ  pycall-node:   Node ‚Üí stdin write ‚Üí Python ‚Üí stdout read   (~1ms per call)
```

|  | Feature | Description |
|--|---------|-------------|
| ‚ö° | **Native async/await** | Call `@expose`'d Python functions like local JS async functions |

```python
# my_model.py

@expose
```

**2 ‚ÄĒ Call it from Node.js**

```typescript
import { Bridge } from 'pycall-node';

await bridge.ready();

const result = await bridge.call('predict', { input: [1, 2, 3] });
console.log(result);
// ‚Üí { label: 'cat', confidence: 0.97 }

await bridge.destroy();
```

No servers. No config files. No boilerplate.

## ūüďĖ API Reference

### `new Bridge(options)`

Spawns the Python subprocess immediately on construction.

```typescript
const bridge = new Bridge({
  pythonScript: './script.py',     // ‚úÖ required  ‚ÄĒ path to your Python file
  pythonPath:   'python3',         // ‚öôÔłŹ optional  ‚ÄĒ python executable (default: 'python3')
  timeout:       30_000,           // ‚öôÔłŹ optional  ‚ÄĒ default call timeout ms (default: 30000)
  maxRestarts:   3,                // ‚öôÔłŹ optional  ‚ÄĒ max crash restarts (default: 3)
  env: { MY_API_KEY: 'secret' },   // ‚öôÔłŹ optional  ‚ÄĒ extra env vars for Python
  cwd: '/path/to/project',         // ‚öôÔłŹ optional  ‚ÄĒ working directory for Python
});
```

<br />

### `bridge.ready() ‚Üí Promise<void>`

Resolves once Python sends its `__ready__` handshake. Always `await` this before calling.

```typescript
await bridge.ready();
// ‚úÖ Python subprocess is up and accepting calls
```

<br />

### `bridge.call(fn, ...args) ‚Üí Promise<unknown>`

Two calling conventions supported:

```typescript
// Variadic args
const result = await bridge.call('predict', inputData);

// Explicit args array + per-call timeout override
const result = await bridge.call('predict', [inputData], { timeout: 5_000 });
```

<br />

### `bridge.callWithKwargs(fn, args, kwargs, options?)`

Pass both positional and keyword arguments to Python:

```typescript
const result = await bridge.callWithKwargs(
  'train_model',
  [dataset],
  { epochs: 10, lr: 0.001, device: 'cuda' },
  { timeout: 120_000 }
);
```

<br />

### `bridge.exposedFunctions ‚Üí readonly string[]`

Function names registered via `@expose` in the Python script. Available after `ready()`.

```typescript
await bridge.ready();
console.log(bridge.exposedFunctions);
// ‚Üí ['predict', 'preprocess', 'train_model']
```

<br />

### `bridge.destroy() ‚Üí Promise<void>`

Graceful shutdown: closes stdin ‚Üí waits for exit ‚Üí force-kills after 2s ‚Üí rejects all pending calls.

```typescript
await bridge.destroy();
```

## ūüŹä Worker Pool

Scale horizontally with `BridgePool` ‚ÄĒ manages N Python workers with automatic round-robin dispatch.

```typescript
import { BridgePool } from 'pycall-node';

const pool = new BridgePool({
  pythonScript: './worker.py',
  size:          4,          // 4 parallel Python processes
  timeout:       30_000,
});

await pool.ready(); // waits for ALL workers

// Round-robin dispatch ‚ÄĒ all 3 calls run in parallel
const [r1, r2, r3] = await Promise.all([
  pool.call('process', chunk1),   // ‚Üí Worker 0
  pool.call('process', chunk2),   // ‚Üí Worker 1
  pool.call('process', chunk3),   // ‚Üí Worker 2
]);

await pool.destroy();
```

**`BridgePoolOptions`**

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `pythonScript` | `string` | required | Path to Python script |
| `size` | `number` | `os.cpus().length` | Number of parallel workers |
| `timeout` | `number` | `30_000` | Default call timeout (ms) |
| `maxRestarts` | `number` | `3` | Max restarts per worker |
| `pythonPath` | `string` | `'python3'` | Python executable |
| `env` | `Record<string, string>` | `‚ÄĒ` | Extra environment variables |
| `cwd` | `string` | `‚ÄĒ` | Working directory |

## ūüõ°ÔłŹ Error Handling

`pycall-node` ships a structured error hierarchy ‚ÄĒ catch broadly or handle each failure mode precisely.

```
PyCallNodeError              ‚Üź base class ‚ÄĒ catch-all safety net
‚Ēú‚ĒÄ‚ĒÄ PyTimeoutError           ‚Üź call() exceeded its deadline
‚Ēú‚ĒÄ‚ĒÄ PyProcessError           ‚Üź subprocess crashed or failed to spawn
‚ĒĒ‚ĒÄ‚ĒÄ PyRuntimeError           ‚Üź Python raised an exception inside your function
```

```typescript
import {
  PyTimeoutError,
  PyProcessError,
  PyRuntimeError,
} from 'pycall-node';

try {
  const result = await bridge.call('predict', data, { timeout: 2_000 });

} catch (err) {

  if (err instanceof PyTimeoutError) {
    // ‚ŹĪÔłŹ Call took too long
    console.error(`Timed out after ${err.timeoutMs}ms`);

  } else if (err instanceof PyRuntimeError) {
    // ūüźć Python raised an exception
    console.error(`${err.pythonType}: ${err.message}`);
    console.error(err.pythonTraceback);   // full Python stack trace

  } else if (err instanceof PyProcessError) {
    // ūüí• Subprocess died unexpectedly
    console.error(`Process exited with code ${err.exitCode}`);
    console.error(err.stderr);            // last stderr output
  }
}
```

**Error properties:**

| Error Class | Extra Properties |
|-------------|-----------------|
| `PyTimeoutError` | `.timeoutMs: number` |
| `PyProcessError` | `.exitCode: number \| null` &nbsp; `.stderr: string` |
| `PyRuntimeError` | `.pythonType: string` &nbsp; `.pythonTraceback: string` |

## ūüĒĄ Auto-Restart & Resilience

When the Python process crashes, `pycall-node` automatically restarts it using **exponential backoff**:

```
delay = min( 2‚ĀŅ √ó 1000ms , 10000ms )

  Crash #1  ‚ĒÄ‚ĒÄ‚Ė∂  wait  1 000ms  ‚ĒÄ‚ĒÄ‚Ė∂  restart ‚úÖ
  Crash #2  ‚ĒÄ‚ĒÄ‚Ė∂  wait  2 000ms  ‚ĒÄ‚ĒÄ‚Ė∂  restart ‚úÖ
  Crash #3  ‚ĒÄ‚ĒÄ‚Ė∂  wait  4 000ms  ‚ĒÄ‚ĒÄ‚Ė∂  restart ‚úÖ
  Crash #4  ‚ĒÄ‚ĒÄ‚Ė∂  ūüíÄ emit 'crash' event, stop retrying
```

```typescript
bridge.on('crash', (err) => {
  console.error('Bridge permanently failed:', err.message);
  process.exit(1);
});
```

> All pending calls during a crash are immediately rejected with `PyProcessError`.

## ūüĒĆ NDJSON Protocol

The internal wire format ‚ÄĒ plain newline-delimited JSON over stdio. No framing, no overhead.

**Node ‚Üí Python `stdin` (request):**
```json
{ "id": "a3f7c2d1-...", "function": "predict", "args": [[1,2,3]], "kwargs": {} }
```

**Python ‚Üí Node `stdout` on success:**
```json
{ "id": "a3f7c2d1-...", "status": "ok", "result": { "label": "cat", "confidence": 0.97 } }
```

**Python ‚Üí Node `stdout` on error:**
```json
{
  "id": "a3f7c2d1-...",
  "status": "error",
  "error": "division by zero",
  "type": "ZeroDivisionError",
  "traceback": "Traceback (most recent call last):\n  File ..."
}
```

Each call registers its `resolve`/`reject` in an in-memory `Map<uuid, PendingCall>`. Dozens of concurrent in-flight calls are fully safe.
